# PeerConnect

A peer study group and resource sharing platform for students. PeerConnect helps students collaborate through study groups, resource sharing, scheduling, and AI-generated quizzes.

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | React 18 + Vite |
| Backend | Node.js + Express |
| Database | MongoDB |
| Auth | JWT |
| Deployment | Docker on EC2 |

## Local Setup

**Prerequisites:** Node.js 20+, npm, MongoDB running locally.

### 1. Clone and configure environment

```bash
git clone https://github.com/danileslie/group2-capstone.git
cd group2-capstone
cp .env.example server/.env
```

Edit `server/.env` and set a real value for `JWT_SECRET` before running.

### 2. Install dependencies

```bash
npm run install:all
```

This installs root, server, and client dependencies in one command.

### 3. Start the development servers

```bash
npm run dev
```

This starts both the Express backend and Vite frontend concurrently:

| Service | URL |
|---|---|
| Backend API | http://localhost:5000 |
| Frontend | http://localhost:5173 |

Individual servers can be started separately:

```bash
npm run dev:server   # backend only
npm run dev:client   # frontend only
```

### 4. Verify the API is running

```bash
curl http://localhost:5000/health
# в†’ {"status":"ok","version":"0.1.0"}
```

## bootstrap.sh

`bootstrap.sh` is designed for setting up a **fresh environment** (e.g. a new EC2 instance or a teammate's machine for the first time). It uses `npm ci` for reproducible installs and seeds the database in non-production environments.

### Prerequisites before running

```bash
# 1. Environment file must exist
cp .env.example server/.env
# Edit server/.env with real values

# 2. Lock files must exist (generated by npm install)
# If missing, run once:
npm install --prefix server
npm install --prefix client
```

### Run the script

```bash
bash bootstrap.sh
```

### What it does

1. Checks that `server/.env` and both `package-lock.json` files exist (fails fast if not)
2. Runs `npm ci` in `server/` and `client/`
3. Skips DB migrations (MongoDB is schema-less)
4. Seeds the database unless `NODE_ENV=production`

### Skip seeding (production)

```bash
NODE_ENV=production bash bootstrap.sh
```

## Environment Variables

All variables live in `server/.env`. Copy `.env.example` to create it:

```bash
cp .env.example server/.env
```

| Variable | Default | Description |
|---|---|---|
| `PORT` | `5000` | Port the Express server listens on |
| `NODE_ENV` | `development` | Environment (`development` / `production`) |
| `MONGO_URI` | `mongodb://localhost:27017/peerconnect` | MongoDB connection string |
| `JWT_SECRET` | *(must be set)* | Secret key for signing JWTs вЂ” use a long random string |
| `JWT_EXPIRES_IN` | `7d` | JWT token expiry duration |
| `CLIENT_URL` | `http://localhost:5173` | Allowed CORS origin (frontend URL) |

**Never commit `server/.env` to git.** It is already listed in `.gitignore`.

## CI Pipeline

Pull requests targeting `main` or `dev` automatically run the CI workflow defined in `.github/workflows/ci.yml`:

| Job | What it does |
|---|---|
| **lint** | Runs `npm run lint` in server and client |
| **test** | Runs `npm test` in server and client |
| **build** | Runs `vite build` in client |
| **health-check** | Starts the server and asserts `GET /health` returns 200 |

Each job must pass before the next begins.

## EC2 Deployment

Deployment uses Docker on an EC2 instance (Ubuntu 22.04). Docker configuration is in progress.

**Planned deployment steps:**

1. SSH into the EC2 instance
2. Clone the repository
3. Copy and configure `server/.env` with production values (`NODE_ENV=production`, real `MONGO_URI`, strong `JWT_SECRET`, public `CLIENT_URL`)
4. Run `NODE_ENV=production bash bootstrap.sh`
5. Start services via Docker Compose

**Production environment notes:**
- Set `NODE_ENV=production` вЂ” this prevents the seed script from running
- `CLIENT_URL` must match the domain or public IP serving the frontend
- `JWT_SECRET` must be a securely generated random string (minimum 32 characters)
- MongoDB should be hosted on Atlas or a dedicated instance, not localhost

This section will be updated when the Dockerfile and docker-compose.yml are added.
